Trinkkets SaaS

A production SaaS platform that analyses millions of marketplace listings to help Etsy sellers identify profitable products and emerging markets.

Overview

The platform is powered by a data pipeline and machine learning models that transform large volumes of Etsy marketplace data into product insights, demand indicators, and market classifications.

Problem

How can Etsy sellers know what Products & Markets will do well on the Etsy platform?

Deliverable

A web service running a data model that categorises and recommends products/Etsy categories based on millions of real sales records.

Tech stack
  • Data Pipelines: Python (asyncio, aiohttp, requests)
  • Machine Learning / NLP: scikit-learn
  • Database: PostgreSQL, SQLAlchemy, Psycopg2
  • Backend: Node.js, Express
  • Frontend: Bootstrap
  • Deployment: Render

Key Features

Product Search

The product search engine analyses millions of Etsy listings to identify products with strong demand signals and lower marketplace competition. A data pipeline aggregates marketplace listings and applies statistical and machine learning techniques to rank products based on supply rarity and demand growth. This allows sellers to quickly discover products that are both popular and underserved in the marketplace.

Market Search

Market search aggregates product-level data to analyse entire Etsy categories. Markets are classified into segments such as Hot, Hidden Gem, Saturated, and Declining, based on their growth rate and competitive intensity. This allows sellers to identify markets where demand is growing but competition remains manageable.

Architecture

Flow
1. Data Ingestion
Collect marketplace data from the Etsy API and store raw records in PostgreSQL.
2. Data Processing Pipeline
Clean, transform and structure the raw data into analysis-ready datasets.
3. Feature Engineering & Modelling
Apply NLP and machine learning techniques to identify product demand signals and market characteristics.
4. Database Layer
Processed outputs update analytical views and summary tables in PostgreSQL.
5. Application Layer
The web platform queries the database to power product search and market analytics features.
6. Deployment & Automation
Continuous deployment pipeline running on Render with automated data updates and platform availability.
